# React和Vue在技术层面有哪些区别?

## 简短回答

React 是 **JS 拥趸**，Vue 是 **模版爱好者**。
React 强调 **Immutable (不可变)** 和 **手动挡**，你需要显式告诉它怎么更新 (setState)，一切都是 JS，灵活性极高；
Vue 强调 **Mutable (可变)** 和 **自动挡**，你只管改数据，它自动收集依赖并更新，上手更简单，内置功能更全。
殊途同归，现代 Vue3 的 Composition API 和 React Hooks 在逻辑复用上已经非常像了。

---

## 1. 组件定义风格

**React (Hooks + JSX)**
JSX 让你在 JS 里写 HTML，拥有 JS 的全部能力 (map, filter)。

```jsx
function Counter() {
  const [count, setCount] = useState(0); // 显式状态管理
  return (
    // 必须调用 setCount 传入新值，不能直接 count++
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  ); 
}
```

**Vue 3 (Composition API + Template)**
Template 将 HTML 结构分离，更符合传统网页开发习惯。

```html
<template>
  <!-- 直接修改，自动更新 -->
  <button @click="count++">{{ count }}</button>
</template>

<script setup>
import { ref } from 'vue';
const count = ref(0); // 响应式对象
</script>
```

## 2. 响应式原理差异

*   **Vue (Proxy)**: 利用 ES6 Proxy 劫持对象。当你 `count.value = 1` 时，Setter 拦截并触发更新。它知道确切的依赖关系，所以不需要像 React 那样手动优化 (useMemo)。
*   **React (Comparison)**: 每次更新都是重新执行整个组件函数。它通过比较新旧 Virtual DOM 来决定更新哪里。所以需要 `React.memo` 来防止不必要的重复渲染。

## 3. 逻辑复用

*   **React**: Custom Hooks (`useMyHook`)。
*   **Vue**: Composables (`useMyComposable`)。
*   *结论*：Vue 3 的 Composition API 深受 React Hooks 启发，解决了 Vue 2 Mixin 命名冲突和来源不清的问题。
