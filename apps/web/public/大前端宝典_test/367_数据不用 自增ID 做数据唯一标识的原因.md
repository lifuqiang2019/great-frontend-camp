# 数据不用 自增ID 做数据唯一标识的原因

1.  **分布式 系统的唯一性：** 非自增ID而使用如 UUID 作为唯一标识，对于分布式系统中的唯一性要求非常重要。
2.  **避免ID泄漏：** 自增ID可能泄漏系统的数据量和增长速度。使用 UUID 可以避免这种潜在的信息泄露。
3.  **数据合并：** 如果您需要合并来自多个不同 数据源 的数据， UUID 可以更容易地确保不同数据源的标识不会冲突。
4.  **隐私和安全：** 使用自增ID时，攻击者可以根据ID的顺序猜测到数据记录的创建时间。 UUID 不会暴露这些信息。
5.  **数据库迁移：** 数据库迁移是可保证数据不改变。
6.  **避免数据库锁定：** 在高并发环境中，使用自增ID时，数据库的锁定可能会成为瓶颈。 UUID 可以减少锁定争用。
